26 /* In order to use the library, you need to define SDA_PIN, SCL_PIN,
27  * SDA_PORT and SCL_PORT before including this file. Have a look at
28  * http://www.arduino.cc/en/Reference/PortManipulation for finding out
29  * which values to use. For example, if you use digital pin 3 (corresponding
30  * to PD3) for SDA and digital pin 13 (corresponding to PB5)
31  * for SCL on a standard Arduino,
32  * you have to use the following definitions:
33  * #define SDA_PIN 3
34  * #define SDA_PORT PORTD
35  * #define SCL_PIN 5
36  * #define SCL_PORT PORTB
37  *
38  * Alternatively, you can define the compile time constant I2C_HARDWARE,
39  * in which case the TWI hardware is used. In this case you have to use
40  * the standard SDA/SCL pins (and, of course, the chip needs to support
41  * this).
42  *
43  * You can also define the following constants (see also below):
44  ' - I2C_PULLUP = 1 meaning that internal pullups should be used
45  * - I2C_CPUFREQ, when changing CPU clock frequency dynamically
46  * - I2C_FASTMODE = 1 meaning that the I2C bus allows speeds up to 400 kHz
47  * - I2C_SLOWMODE = 1 meaning that the I2C bus will allow only up to 25 kHz
48  * - I2C_NOINTERRUPT = 1 in order to prohibit interrupts while
49  * communicating (see below). This can be useful if you use the library
50  * for communicating with SMbus devices, which have timeouts.
51  * Note, however, that interrupts are disabled from issuing a start condition
52  * until issuing a stop condition. So use this option with care!
53  * - I2C_TIMEOUT = 0...10000 msec in order to return from the I2C functions
54  * in case of a I2C bus lockup (i.e., SCL constantly low). 0 means no timeout.
55  * - I2C_MAXWAIT = 0...32767 number of retries in i2c_start_wait. 0 means never stop.
56  */
